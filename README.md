# Model Serving with Docker and FastAPI

End-to-end ML model serving pipeline using a Linear Regression model trained on the California Housing dataset, exposed via FastAPI and containerized with Docker.

## Folder Structure

```
infra01-model-serving-docker/
├── data/                  # raw data and sample inputs
├── models/                # serialized model (generated by train.py)
├── src/
│   ├── app/
│   │   └── main.py        # FastAPI application
│   └── train.py           # training pipeline
├── Dockerfile
├── docker-compose.yml
└── requirements.txt
```

## How to Run

**Prerequisites:**
- [Docker Desktop](https://www.docker.com/products/docker-desktop/)
- [California Housing dataset](https://www.kaggle.com/datasets/shibumohapatra/house-price) — save as `data/california-housing.csv`

**Steps:**
1. Clone the repository
2. Run the training script to generate the model: `python src/train.py`
3. Start the container: `docker compose up --build`
4. Test the endpoints

## Endpoints

- `GET /health` — returns service status
- `POST /predict` — returns median house value prediction

**Payload example:**
```json
{
  "longitude": -124.23,
  "latitude": 40.54,
  "housing_median_age": 52,
  "total_rooms": 2694,
  "total_bedrooms": 453.0,
  "population": 1152,
  "households": 435,
  "median_income": 3.0806,
  "ocean_proximity": "NEAR OCEAN"
}
```

## Running the tests

To run the tests, run `pip install -r requirements-test.txt`
So, run `pytest` at the terminal

## CI/CD Pipeline ![CI](https://github.com/drdhenrique/infra01-model-serving-docker/actions/workflows/ci.yml/badge.svg)

Every push triggers an automated pipeline with three jobs:

- **test** — trains the model and runs pytest, including a validation gate (R² ≥ 0.5)
- **build** — builds the Docker image to validate the Dockerfile
- **push** — pushes the image to Docker Hub (runs only on main branch)

If any job fails, the subsequent jobs do not run.
